# Architecture diagrams

Source of truth for the Rust + AWS SaaS architecture visuals.

## Files

- `architecture.py` — Python [diagrams](https://diagrams.mingrammer.com/) script generating:
  - `architecture_main.png` — request path + data plane (C4 L2).
  - `architecture_observability.png` — telemetry topology.
- `*.png` — rendered outputs (regenerated by CI, safe to commit).

## Rendering locally

Install dependencies once:

```bash
brew install graphviz                # macOS (or: sudo apt-get install graphviz)

cd docs/diagrams
python3 -m venv .venv                # isolated env (PEP 668-safe)
.venv/bin/pip install diagrams
```

Regenerate the PNGs:

```bash
cd docs/diagrams
.venv/bin/python architecture.py
```

## When to update

- Any PR that adds/removes an AWS service in the data path.
- Any PR that changes trust boundaries (SG, IAM, VPC endpoints).
- Any PR that changes the observability stack.

`CODEOWNERS` should require review from a platform engineer when files under
`docs/diagrams/` change, to avoid diagram rot.
